Class: RtcMedia

RtcMedia

Constructs a new RtcMedia instance

Create an Offer

  • Get Stream from getUserMedia
  • Set Stream
  • pc.onnegotiationneeded
  • pc.createOffer
  • pc.setLocalDescription(offer)
  • Send SDP --> signalingChannel.send(JSON.stringify({ "sdp": pc.localDescription }));

Create an Answer

  • Get SDP from remote site
  • Creat (desc)
  • pc.setRemoteDescription(desc)
  • pc.createAnswer()
  • pc.setLocalDescription(answer)
  • Send SDP --> signalingChannel.send(JSON.stringify({ "sdp": pc.localDescription }))

(static) _chn_pc_SetLocalStreamAndTestNegotiation(mySelfPromiseData) → {Promise(PromiseData)}

Generic version. Calls RtcMedia._chn_pc_TestNegotiationNeeded AND RtcMedia._chn_pc_SetLocalStream together. This sets up RtcMedia._chn_pc_TestNegotiationNeeded first (but does not wait for it), then sets up RtcMedia._chn_pc_SetLocalStream. RtcMedia._chn_pc_SetLocalStream SHOULD trigger a peerConn.negotiationNeeded event which will resolve the RtcMedia._chn_pc_TestNegotiationNeeded. If RtcMedia._chn_pc_SetLocalStream does not trigger a peerConn.negotiationNeeded event THEN RtcMedia._chn_pc_TestNegotiationNeeded times out.

To determine if RtcMedia._chn_pc_TestNegotiationNeeded timesOut should trigger a Resolve or Reject, the RtcMedia#negotiationNeededTimeOutReject is queried.

With this funciton call RtcMedia#negotiationNeededTimeOutReject should be set by developer or else the default (or previously set) value will be used.

It is MUCH better to use one of these:

  • RtcMedia._chn_pc_SetLocalStreamAndTestNegotiationResolve or
  • RtcMedia._chn_pc_SetLocalStreamAndTestNegotiationReject,
as they set those values before calling this function.
